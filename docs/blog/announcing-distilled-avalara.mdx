Sales-tax code sits at an uncomfortable intersection: business-critical, integration-heavy, and unforgiving of silent data-shape mistakes. [`@kevinmichaelchen/distilled-avalara`](https://www.npmjs.com/package/@kevinmichaelchen/distilled-avalara) brings Avalara AvaTax REST v2 into Effect with 470 generated operations and provider-aware runtime policy.

## AvaTax, at a glance

[Avalara AvaTax](https://www.avalara.com/us/en/products/sales-and-use-tax/avatax.html) automates sales and use tax calculations and related compliance workflows. Its REST v2 surface covers tax calculation, transactions, addresses, exemptions, certificates, filings, reports, jurisdictions, nexus, and account configuration. Avalara publishes [developer documentation and SDKs](https://developer.avalara.com/products/avatax/sdk), including its official [`AvaTax-REST-V2-JS-SDK`](https://github.com/avadev/AvaTax-REST-V2-JS-SDK).

## Why Effect and Distilled?

Tax calculations deserve an explicit network boundary. Generated schemas describe request and response shapes and carry the exact HTTP traits used on the wire. The AvaTax Protocol owns authentication, vendor error envelopes, and retry hints, while Effect keeps those failures and transport errors in a composable error channel.

That becomes more valuable as a flow grows from a rate estimate into transaction creation, adjustment, commitment, refunding, and reconciliation: the whole workflow shares one retry, tracing, and failure model.

## How we made the SDK

<Steps>
  <Step title="Mirror Avalara's Swagger">A dedicated repository stores a formatted-but-semantically-unchanged copy of the official AvaTax REST v2 document.</Step>
  <Step title="Pin the source commit">The implementation consumes that repository as a submodule, making every release traceable to one immutable contract.</Step>
  <Step title="Generate and review">The shared Protocol generator emits 470 operations grouped into resource services, committed so changes remain visible in review.</Step>
  <Step title="Model Avalara policy explicitly">The handwritten Protocol supports bearer tokens or account ID and license key authentication, the required client-identification header, vendor errors, and rate-limit guidance.</Step>
</Steps>

```ts
import * as Effect from "effect/Effect";
import * as Layer from "effect/Layer";
import * as FetchHttpClient from "effect/unstable/http/FetchHttpClient";
import {
  CredentialsFromEnv,
  Services,
} from "@kevinmichaelchen/distilled-avalara";

const AvaTaxLive = Layer.mergeAll(FetchHttpClient.layer, CredentialsFromEnv);
```

:::tip[One layer, the whole AvaTax API]
With `AvaTaxLive` in scope, you can calculate tax, validate addresses, create and reconcile transactions, manage nexus, exemptions and certificates, inspect jurisdictions, and automate filings and reports.
:::

### Estimate the rate for an address

Use the address endpoint for a quick jurisdictional estimate before constructing a transaction.

```ts
const estimate = Services.tax_content.taxRatesByAddress({
  line1: "100 Market Street",
  city: "Philadelphia",
  region: "PA",
  postalCode: "19106",
  country: "US",
});

const estimateAddressRates = estimate.pipe(Effect.provide(AvaTaxLive));
```

### Calculate a sales transaction

Create an uncommitted sales order so AvaTax can account for nexus, sourcing, product taxability, exemptions, and line-level addresses.

```ts
const calculateSalesTransaction = Services.transactions.createTransaction({
  body: {
    companyCode: "DEFAULT",
    type: "SalesOrder",
    date: "2026-07-15",
    customerCode: "CUST-1007",
    commit: false,
    lines: [{
      number: "1",
      quantity: 2,
      amount: 199.98,
      taxCode: "P0000000",
      addresses: {
        shipFrom: { line1: "1 Warehouse Way", city: "Newark", region: "NJ", postalCode: "07102", country: "US" },
        shipTo: { line1: "100 Market Street", city: "Philadelphia", region: "PA", postalCode: "19106", country: "US" },
      },
    }],
  },
}).pipe(Effect.provide(AvaTaxLive));
```

### Retrieve a transaction for reconciliation

Look up a previously calculated document by the identifiers stored in an order or accounting system.

```ts
const retrieveTransaction = Services.transactions.getTransactionByCode({
  companyCode: "DEFAULT",
  transactionCode: "ORDER-10492",
  documentType: "SalesOrder",
  $include: "Lines,Details",
}).pipe(Effect.provide(AvaTaxLive));
```

All operation functions use lower camelCase; their input and output schemas use PascalCase.
