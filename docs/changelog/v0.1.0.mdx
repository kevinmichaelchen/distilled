The first Distilled release establishes the standalone software-factory pattern and ships Jira, GitHub, and Slack SDKs.

## Packages

- [`@kevinmichaelchen/distilled`](https://www.npmjs.com/package/@kevinmichaelchen/distilled) — shared Effect Protocol engine and OpenAPI generator
- [`@kevinmichaelchen/distilled-jira`](https://www.npmjs.com/package/@kevinmichaelchen/distilled-jira) — Effect-native Jira SDK
- [`@kevinmichaelchen/distilled-github`](https://www.npmjs.com/package/@kevinmichaelchen/distilled-github) — Effect-native GitHub SDK
- [`@kevinmichaelchen/distilled-slack`](https://www.npmjs.com/package/@kevinmichaelchen/distilled-slack) — Effect-native Slack SDK

## Added

- Shared Effect 4 Protocol engine with trait-driven HTTP operations
- Deterministic OpenAPI 2/3 generation with RFC 6902 patch envelopes
- Jira source mirror and [`@kevinmichaelchen/distilled-jira`](https://www.npmjs.com/package/@kevinmichaelchen/distilled-jira)
- GitHub source bundle mirror and [`@kevinmichaelchen/distilled-github`](https://www.npmjs.com/package/@kevinmichaelchen/distilled-github)
- Slack TypeScript-source generator and [`@kevinmichaelchen/distilled-slack`](https://www.npmjs.com/package/@kevinmichaelchen/distilled-slack)
- Dual source-first Bun and compiled Node exports
- GitHub Actions OIDC trusted publishing with npm tokens disabled
- Blume documentation, local search, RSS, raw Markdown mirrors, and GitHub Pages deployment

## Verified

All four npm packages were installed into a clean consumer directory and imported successfully with standard Node ESM. The SDK release workflows also completed through OIDC without npm tokens.

> **Patch note:** Jira and GitHub received `0.1.1` releases immediately after launch to make one relative source import follow the same explicit `.ts` extension pattern used by Alchemy. Slack already followed that pattern at `0.1.0`.
