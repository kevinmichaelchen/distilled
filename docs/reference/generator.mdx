`generateFromOpenAPI` reads an OpenAPI 2 or 3 document, applies documented patch envelopes, and emits resource-grouped services using the shared Protocol operation contract.

<AutoTypeTable path="./scripts/generate-openapi.ts" name="GeneratorConfig" />

## Minimal invocation

```ts
import { generateFromOpenAPI } from "@kevinmichaelchen/distilled/openapi/generate";

const coverage = generateFromOpenAPI({
  specPath: "specs/provider/specs/openapi.json",
  patchDir: "patches",
  outputDir: "src/services",
  protocolName: "ProviderProtocol",
  operationErrorType: "ProviderOpError",
  operationContextType: "ProviderOpContext",
});
```

The generated directory should be committed and checked for drift after regeneration.

## Output contract

Generation is all-or-nothing. Operation failures are collected and reported together, and the existing output directory is left intact. A successful run replaces the directory as one complete projection, so stale service modules cannot survive a source change.

Every successful output contains:

- generated resource service modules and a namespace `index.ts` barrel;
- `coverage.json`, a deterministic record of the specification identity, applied patches, deprecations, unsupported operations, failures, and totals by HTTP method; and
- lazily constructed request and response schemas using `S.suspend`; and
- query, path, header, body, method, and response traits that retain exact wire names.

Generated inputs keep path, query, and header members at the top level. A JSON
request body is nested under `body` and marked with `T.HttpBody()`, matching the
Protocol operation shape used by Alchemy.

The function returns the same coverage value written to disk:

<AutoTypeTable path="./scripts/generate-openapi.ts" name="GenerationCoverage" />

Treat a nonzero failure count as a broken generation run, not as partial success. Review coverage changes alongside generated source changes, especially when the upstream specification adds or removes operations.
