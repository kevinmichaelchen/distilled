The shared package implements the provider-neutral half of every SDK. Provider
packages supply credentials and a Protocol layer; generated services supply
trait-annotated schemas and explicit operations.

## Export map

| Subpath | Purpose |
| --- | --- |
| `/api` | `Protocol`, `OperationMethod`, `make`, and `makePaginated` |
| `/trait` | HTTP, wire-name, response, and typed-error annotations |
| `/protocol-http` | Trait-driven request construction and response helpers |
| `/errors` | Shared typed HTTP and configuration errors |
| `/category` and `/error-category` | Cross-provider semantic error categories |
| `/retry` and `/retry-after` | Retry policies and server delay parsing |
| `/pagination` | Page, cursor, token, and single-result streams |
| `/schema` | Lightweight generated-code surface over Effect Schema |
| `/json-patch` | Source-model patch support |
| `/openapi/generate` | OpenAPI-to-Protocol generator entry point |

## Operation contract

An operation is both callable and yieldable:

```ts
import { Services } from "@kevinmichaelchen/distilled-github";

const repository = yield* Services.repos.reposGet({
  owner: "effect-ts",
  repo: "effect",
});

const getRepository = yield* Services.repos.reposGet;
const sameRepository = yield* getRepository({
  owner: "effect-ts",
  repo: "effect",
});
```

For OpenAPI providers, the operation context includes provider credentials and
the Effect HTTP transport. Slack instead provides its Protocol through the
official typed `WebClient`. The error channel names the provider-wide, declared
operation, and transport failures available to that SDK.
