The AvaTax SDK pins Avalara's official REST v2 Swagger document and keeps authentication, errors, retries, and the `X-Avalara-Client` header in its provider Protocol.

## Install

<CodeGroup>

```bash npm
npm install @kevinmichaelchen/distilled-avalara effect
```

```bash pnpm
pnpm add @kevinmichaelchen/distilled-avalara effect
```

```bash bun
bun add @kevinmichaelchen/distilled-avalara effect
```

</CodeGroup>

## Authentication

| Variable | Purpose |
| --- | --- |
| `AVALARA_BEARER_TOKEN` | Preferred bearer credential when available |
| `AVALARA_ACCOUNT_ID` + `AVALARA_LICENSE_KEY` | AvaTax account credential pair |
| `AVALARA_BASE_URL` | Optional endpoint; use `https://sandbox-rest.avatax.com` for sandbox |

## Quickstart

Estimate the combined rate for an address:

```ts
import * as Effect from "effect/Effect";
import * as Layer from "effect/Layer";
import * as FetchHttpClient from "effect/unstable/http/FetchHttpClient";
import {
  CredentialsFromEnv,
  Services,
} from "@kevinmichaelchen/distilled-avalara";

const AvaTaxLive = Layer.mergeAll(FetchHttpClient.layer, CredentialsFromEnv);

const program = Services.tax_content.taxRatesByAddress({
  line1: "100 Market Street",
  city: "Philadelphia",
  region: "PA",
  postalCode: "19106",
  country: "US",
});

const rates = await Effect.runPromise(Effect.provide(program, AvaTaxLive));
```

:::tip[One layer, the whole AvaTax API]
The same layer can validate addresses, create and reconcile transactions, manage nexus, exemptions and certificates, inspect jurisdictions, and automate filings and reports. [See three common AvaTax workflows](/blog/announcing-distilled-avalara#code-examples).
:::
