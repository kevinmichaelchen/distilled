The OpenSearch SDK pins the official client-generation contract and keeps cluster addressing, authentication, error mapping, and retry policy in the handwritten provider layer.

</CodeGroup>

## Authentication

Set `OPENSEARCH_URL` and either `OPENSEARCH_USERNAME` with `OPENSEARCH_PASSWORD`, or `OPENSEARCH_TOKEN`. A deliberately unsecured local cluster can use only `OPENSEARCH_URL`.

Amazon OpenSearch Service uses AWS Signature Version 4. Supply a signing HTTP client layer for that deployment model; the core package deliberately does not import a cloud-provider SDK.
