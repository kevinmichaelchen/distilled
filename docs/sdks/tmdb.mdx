The TMDB SDK projects the official TMDB v3 OpenAPI document into Effect operations while keeping authentication, error decoding, retry hints, and configurable API addressing in a handwritten Protocol.

## Authentication

Create a TMDB API Read Access Token and set `TMDB_API_READ_ACCESS_TOKEN`. The SDK sends it as a redacted bearer token. `TMDB_API_URL` can override the default endpoint.

The SDK deliberately does not read `TMDB_API_KEY`. TMDB also supports an API key through the separate `api_key` query parameter, but this package currently exposes only the bearer-token path so its credential behavior stays unambiguous.

The package is developed and tested with Effect `4.0.0-beta.99`; its peer range also accepts compatible Effect 4 releases from beta 97 onward.

## Quickstart

Fetch the details for *Fight Club*:

```ts
import * as Effect from "effect/Effect";
import * as Layer from "effect/Layer";
import * as FetchHttpClient from "effect/unstable/http/FetchHttpClient";
import {
  CredentialsFromEnv,
  Services,
} from "@kevinmichaelchen/distilled-tmdb";

const TmdbLive = Layer.mergeAll(
  FetchHttpClient.layer,
  CredentialsFromEnv,
);

const movie = await Effect.runPromise(
  Services.movie.movieDetails({ movie_id: 550 }).pipe(
    Effect.provide(TmdbLive),
  ),
);
```

The same layer supports search, discovery, trending media, credits, people, television, watch providers, lists, and account operations. Direct resource subpaths are also available, such as `@kevinmichaelchen/distilled-tmdb/movie`.
